Deconstructing Palantir Ontology: When an Ontology Moves From Describing the World to Driving It 解构 Palantir 本体论：当本体从描述世界转向驱动世界

2026-07-18 · 12 min read阅读约 12 分钟 AI & Machine LearningEnterprise AIData Platforms 人工智能与机器学习企业 AI数据平台

An ontology used to be a description — a shared vocabulary that let teams agree on what a "customer" or a "shipment" was. Palantir's move is to make the ontology act: the same objects that describe the business also carry the functions that change it. 本体过去只是一种描述——一套共享词汇，让团队对"客户"或"一批货"是什么达成一致。Palantir 的转变在于让本体产生行动：那些描述业务的对象，同时也承载着改变业务的函数。

Why enterprises need a third kind of ontology为什么企业需要第三种本体论

The "runaway" nature of large models forced an uncomfortable question: if an agent can write back to the system of record, what stops it? The answer is not a better prompt — it is an ontology that encodes permissions, side effects, and reversibility as first-class properties of every object. 大模型的"失控"特性迫使我们面对一个不适的问题：如果智能体可以写回记录系统，谁来约束它？答案不是更好的提示词，而是一套把权限、副作用与可逆性编码为每个对象一等属性的本体。

This is where description ends and action begins. A read-only graph tells you the world exists; an actionable ontology tells you which changes are safe, who may make them, and how to undo them. 描述到此为止，行动由此开始。只读图谱告诉你世界"是什么"；可执行的本体则告诉你哪些改动是安全的、谁可以做出改动，以及如何撤销。
