# 如何测试和部署 Hexo 博客

Last updated: 2026-05-31

## 当前结构

这个仓库现在是一个已经落地的 Hexo 站点，不再是“从零安装 Matery 主题”的通用教程。

当前关键目录：

```text
_config.yml                站点配置，当前主题为 xinhe-site
source/_posts/             文章
source/writing/            写作页
source/technical-notes/    技术笔记页
source/projects/           项目页
source/about/              About & Contact 页
source/about-zh/           个人信息和联系方式页
source/contact/            兼容跳转页
themes/xinhe-site/         当前生效主题
```

## 前置条件

- Node.js
- Yarn
- Git

检查版本：

```bash
node -v
yarn --version
git --version
```

安装依赖：

```bash
yarn install
```

## 本地预览

推荐流程：

```bash
yarn clean
yarn server
```

访问：

```text
http://localhost:4000
```

如果只想检查能否成功生成静态文件：

```bash
yarn build
```

## 当前页面模型

主导航固定为五项：

```text

```

对应中文页面：

```text
/home-zh/

```

注意：

- `/contact/` 只保留为兼容入口，不再是独立主导航页面。
- 英文页和中文页分路由，不要在同一个生产页面里混排双语正文。
- 中文 About 页可见名称统一为“个人信息和联系方式”。

## 主题维护

当前主题是 `themes/xinhe-site/`。

维护原则：

- 新的站点样式、布局、脚本修改都优先放在 `themes/xinhe-site/`。
- `themes/matery/` 不是运行时主题；如果仍出现在工作区，应视为可清理的历史副本。
- 如果文档或代码里还写着 `themes/matery/` 是活动主题，应该视为过时信息。
- Header/nav 不应使用 Materialize 默认红色；主题、语言、主导航样式由 `xinhe-site.css` 控制。

当前重点文件：

```text
themes/xinhe-site/_config.yml
themes/xinhe-site/layout/layout.ejs
themes/xinhe-site/layout/_partial/navigation.ejs
themes/xinhe-site/layout/_partial/footer.ejs
themes/xinhe-site/source/css/xinhe-site.css
themes/xinhe-site/source/js/xinhe-site.js
```

## Mermaid、公式、代码高亮

当前站点依赖 `_config.yml` 中已有配置，不需要再按旧教程重装 Matery。

- Mermaid：使用 `{% mermaid %} ... {% endmermaid %}`
- 数学公式：使用 KaTeX 预渲染
- 代码高亮：使用 Prism

示例：

```markdown
行内公式：$E=mc^2$

行间公式：
$$
\frac{\partial u}{\partial t} = h^2 \left( \frac{\partial^2 u}{\partial x^2} + \frac{\partial^2 u}{\partial y^2} + \frac{\partial^2 u}{\partial z^2} \right)
$$
```

## 部署

部署前先确认本地生成成功：

```bash
yarn clean
yarn build
```

部署到 GitHub Pages：

```bash
yarn clean
yarn deploy
```

约定：

- `source` 分支保存源码
- `main` 分支保存生成后的静态文件

## 常见检查

发布前至少检查：

1. `yarn build` 是否通过
2. 主导航五个页面是否都能打开
3. `/contact/` 是否仍然正确跳到 About & Contact
4. 中英文页面切换是否可达
5. KaTeX、Mermaid、Prism 是否正常渲染
6. 是否误写入不应公开的客户名、内部架构、私有指标

快速搜索敏感词：

```bash
rg -n "client|客户|内部|private|confidential|metric|指标|case study|案例" source themes/xinhe-site
```

## 多设备同步

常规流程：

```bash
git checkout source
git pull
yarn install
yarn server
```

完成修改后：

```bash
git add .
git commit -m "更新站点内容"
git push origin source
```

如果要上线，再执行：

```bash
yarn deploy
```
